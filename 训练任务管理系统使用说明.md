# 训练任务管理系统使用说明

## 概述

训练任务管理系统解决了多个训练任务同时触发导致显卡OOM的问题。系统采用队列机制，确保训练任务串行执行，避免资源冲突。

## 系统架构

\`\`\`
训练任务管理系统
├── 训练任务管理器.py      # 核心任务管理器
├── 训练目录监听.py        # 文件监听和自动触发
├── 手动训练工具.py        # 手动添加训练任务
├── 任务状态查看器.py      # 任务状态监控
└── 算法端.py             # 训练算法实现
\`\`\`

## 核心特性

- ✅ **串行执行**: 确保同时只有一个训练任务运行，避免显卡OOM
- ✅ **任务队列**: 自动管理等待中的训练任务
- ✅ **重复检测**: 避免相同任务重复添加
- ✅ **状态监控**: 实时查看任务执行状态
- ✅ **错误处理**: 完善的异常处理和日志记录
- ✅ **持久化**: 任务信息可保存到文件

## 使用方法

### 1. 自动训练（文件监听模式）

启动文件监听服务，当数据满足条件时自动添加训练任务：

\`\`\`bash
python 训练目录监听.py
\`\`\`

系统会监听 `output` 目录，当某个产品文件夹的OK图片≥130张、NG图片≥50张时：
1. 自动整理数据到 `data` 目录
2. 添加训练任务到队列
3. 串行执行训练

### 2. 手动添加训练任务

#### 交互式模式（推荐）
\`\`\`bash
python 手动训练工具.py -i
\`\`\`

按提示输入参数，系统会验证数据目录并添加任务。

#### 命令行模式
\`\`\`bash
# 基本用法
python 手动训练工具.py /path/to/data

# 完整参数
python 手动训练工具.py /path/to/data \
    --name "我的训练任务" \
    --backbone resnet18 \
    --layers layer1,layer2,layer3 \
    --coreset-ratio 0.1 \
    --neighbors 9
\`\`\`

**参数说明：**
- `data_path`: 数据目录路径（必需包含train/OK, test/OK, test/NG子目录）
- `--name`: 任务名称
- `--backbone`: 骨干网络（resnet18, resnet34, resnet50, wide_resnet50_2）
- `--layers`: 特征层，逗号分隔
- `--coreset-ratio`: 核心集采样比例（0-1）
- `--neighbors`: 邻居数量

### 3. 监控任务状态

#### 查看队列状态
\`\`\`bash
python 任务状态查看器.py -q
\`\`\`

#### 查看所有任务
\`\`\`bash
python 任务状态查看器.py -a
\`\`\`

#### 查看特定任务详情
\`\`\`bash
python 任务状态查看器.py -t task_20241225_120000_0001
\`\`\`

#### 持续监控模式
\`\`\`bash
python 任务状态查看器.py -w
\`\`\`

#### 取消等待中的任务
\`\`\`bash
python 任务状态查看器.py -c task_20241225_120000_0001
\`\`\`

#### 保存任务报告
\`\`\`bash
python 任务状态查看器.py -s tasks_report.json
\`\`\`

## 常见问题

### Q: 如何确保任务不会重复执行？
A: 系统会检查相同name和root的任务，如果已存在且状态为PENDING或RUNNING，会跳过重复添加。

### Q: 训练失败了怎么办？
A: 可以通过任务状态查看器查看失败原因，修复问题后重新添加任务。

### Q: 如何停止正在执行的任务？
A: 正在执行的任务无法直接停止，只能等待完成或失败。可以取消等待中的任务。

### Q: 系统重启后任务会丢失吗？
A: 可以使用 `save_tasks_to_file()` 和 `load_tasks_from_file()` 方法持久化任务信息。

### Q: 如何调整队列大小？
A: 队列大小没有限制，但建议根据系统资源合理控制任务数量。

## 日志和调试

系统会输出详细的日志信息，包括：
- 任务添加/执行/完成事件
- 错误信息和堆栈跟踪
- 队列状态变化

日志级别可以通过修改 `logging.basicConfig()` 调整。

## 性能优化建议

1. **合理设置参数**: 根据数据集大小调整 `coreset_sampling_ratio`
2. **选择合适的骨干网络**: 较小的网络训练更快但可能精度较低
3. **监控系统资源**: 确保有足够的内存和显存
4. **定期清理**: 删除不需要的已完成任务记录
